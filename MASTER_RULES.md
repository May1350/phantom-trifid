# MASTER RULES

이 문서는 AI 에이전트가 `phantom-trifid` 프로젝트에서 작업을 수행할 때 반드시 준수해야 하는 마스터 룰입니다.

## 1. 효율적인 브라우저 테스트 (Browser Testing Efficiency)
- **최소 검증 원칙**: 브라우저를 연동하여 테스트할 때는 요청받은 기능이 정상적으로 동작하는지 확인하기 위한 **최소한의 필수 경로**만을 실행합니다.
- **리소스 최적화**: 불필요한 탐색이나 클릭을 지양하여 작업 효율성을 극대화합니다.

## 2. 표준 테스트 계정 (Standard Test Credentials)
검증 작업 시 별도의 요청이 없다면 아래의 계정을 사용합니다.
- **일반 사용자 (Agency)**: `test@gmail.com` / `1111`
- **관리자 (Admin)**: `admin@gmail.com` / `1111`

## 3. 작업 리스크 관리 (Workload & Risk Management)
- **단계적 접근**: 작업의 규모가 커지거나 복잡도가 예상보다 높을 경우, 무리하게 진행하지 않고 작업을 일시 중단합니다.
- **계획 제시**: 즉시 다음 단계의 구체적인 이행 계획을 수립하여 사용자에게 제시하고 승인을 득한 후 진행합니다.

## 4. 포트 사용 및 관리 원칙 (Port Discipline)
- **지정 포트 준수**:
  - **Frontend**: `3000`
  - **Backend**: `3001`
  - **Others**: `3002`
- **우회 금지**: 지정된 포트가 사용 중이거나 연결이 안 될 경우, 임의로 포트를 변경하여 실행하지 않습니다.
- **원인 해결**: 포트 충돌이나 실행 오류 발생 시, 좀비 프로세스를 종료하거나 코드를 수정하여 **반드시 지정된 포트**에서 서비스가 실행되도록 조치합니다.
